## 프롬프트 유형과 선택 이유
```
프롬프트 유형: 템플릿 (Template) / 레시피 (Recipe) 유형 (참조 링크의 10번)

선택 이유:

구조화된 결과물 생성: 이 챌린지의 핵심은 단순한 텍스트 답변이 아닌, 즉시 실행 가능한 HTML/CSS/JS 코드를 생성하는 것입니다. 템플릿 유형은 이처럼 형식이 명확한 결과물을 요청할 때 가장 효과적입니다.

다중 제약 조건 명시: 기획(직관성), 디자인(다크 모드), 운영(데이터 밀집)이라는 3가지의 상충하는 요구사항을 [입력 변수]로 명확히 지정할 수 있습니다.

신속한 프로토타이핑: 템플릿을 사용하여 AI에게 [요구사항 1], [요구사항 2], [요구사항 3]을 만족하는 [HTML 코드], [CSS 코드]를 생성하라고 명확히 지시함으로써, 오후 4시 회의 전까지 결과물을 도출하는 데 가장 적합합니다.
```

## 작성 위해 추가 필요 요건
```
현재 주어진 요구사항("직관적", "트렌디한", "데이터 밀집형")은 매우 추상적입니다. AI가 실제 작동하는 코드를 생성하게 하려면, 다음과 같은 구체적인 정보가 추가로 필요합니다.

핵심 기능 정의: "영상 리뷰 및 승인"을 위해 필수적인 인터랙션 요소 (예: '승인' / '반려' 버튼, '수정 요청' 댓글창 등)

데이터 테이블 명세: 운영팀이 보려는 "데이터 밀집형 테이블"에 포함될 정확한 컬럼명 (예: 영상 ID, 썸네일, 제목, 업로더, 업로드 일시, 현재 상태)

UI 기본 구조: 기획팀의 "직관적인 UI"를 만족시킬 기본 레이아웃 구조 (예: 2단 레이아웃 (좌측 메뉴, 우측 콘텐츠) / 카드형 그리드 레이아웃)

디자인 가이드라인: 디자인팀의 "트렌디한 다크 모드"를 구현할 최소한의 색상 팔레트 (예: 메인 배경색, 기본 텍스트색, 포인트 컬러)

필수 인터랙션: 버튼 클릭 외에 구현되어야 할 핵심 인터랙션 (예: 테이블 행 클릭 시 영상 상세 팝업 노출)
```

```
## 가상 문제 상황
제시해주신 상황(멀티미디어 기업, 업무용 웹페이지 요청, 의견 조율을 위한 프로토타입 작성)을 바탕으로 시나리오 챌린지를 생성했습니다.
수강생이 이 시나리오를 보고 "코딩/프로토타이핑" 관련 프롬프트 작성 능력을 훈련할 수 있도록 설계되었습니다.

🎬 시나리오 챌린지: 의견 조율을 위한 신속한 웹 프로토타입 개발
1. 가상 문제 상황
👤 페르소나: 멀티미디어 콘텐츠 기업 '네오스트림'의 프론트엔드 개발자 (3년 차)
🏙 배경:
귀사의 콘텐츠 운영팀으로부터 "사내 영상 리뷰 및 승인용 웹 대시보드" 제작을 긴급하게 요청받았습니다.
현재 기획서가 완성되지 않은 상태이며, 기획팀은 "직관적인 UI", 디자인팀은 "트렌디한 다크 모드", 운영팀은 "데이터 밀집형 테이블"을 각각 주장하며 의견이 좁혀지지 않고 있습니다.
💥 핵심 문제:
말로만 회의를 진행하다 보니 서로 상상하는 결과물이 달라 의사결정이 지연되고 있습니다.
오늘 오후 4시 회의 전까지, 실제로 작동하는 형태의 결과물이 없으면 프로젝트 착수가 무기한 연기될 위기입니다.
디자인 툴(Figma 등)로 그리기에는 시간이 부족하고, 버튼 클릭 등 인터랙션이 가능한 HTML/CSS 기반의 퍼블리싱 결과물을 보여줘야만 이해관계자들을 설득할 수 있습니다.
🎯 해결 과제:
요구사항(영상 플레이어 영역, 타임스탬프 댓글 기능, 승인/반려 버튼, 다크 모드 스타일)을 포함한 **단일 페이지 웹 프로토타입 코드(HTML, CSS, JS)**를 AI를 통해 즉시 생성하십시오.
이 코드는 회의 시간 내에 즉석에서 수정 요청을 반영할 수 있을 만큼 구조가 명확해야 합니다.
2. 훈련 초점 (힌트)
이 문제는 [코딩 및 프로토타입 생성 / 시각화] 능력을 평가
단순한 텍스트 생성이 아니라, 구체적인 UI/UX 요구사항을 구조화된 언어(코드)로 변환하는 프롬프트 엔지니어링 기술이 필요
```

## 문제 해결 프롬프트
### 🚀 4PM 회의 돌파를 위한 '영상 리뷰 대시보드' 긴급 프로토타입 생성 프롬프트

**To:** (AI 어시스턴트)
**From:** (시니어 프론트엔드 개발자 / PM / UX 디자이너)

당신은 HTML, CSS, JavaScript를 활용하여 신속한 웹 프로토타입 제작에 특화된 시니어 프론트엔드 개발자입니다.

지금부터 3가지의 상충하는 요구사항을 **모두** 만족시키는 **단일 HTML 파일**을 생성해야 합니다. 이 프로토타입은 이해관계자들의 의견 조율을 위한 시각적/인터랙티브 합의점(visual agreement)을 만드는 것이 유일한 목표입니다.

---

### [1] 핵심 요구사항 (Inputs)

1.  **[기획팀] 직관적인 UI:**
    * **해석:** 표준적인 **2단 대시보드 레이아웃**을 적용합니다.
    * **구현:**
        * `좌측`: 250px 너비의 고정된 **사이드바 (LNB)**. "대시보드", "리뷰 대기", "완료된 영상", "설정" 메뉴 목록을 포함합니다.
        * `우측`: **메인 콘텐츠 영역**. 상단에 "리뷰 대기 영상 (5)" 같은 명확한 **페이지 타이틀**이 있어야 합니다.

2.  **[디자인팀] 트렌디한 다크 모드:**
    * **해석:** 완전한 검은색이 아닌, 세련된 **다크 그레이** 기반의 컬러 팔레트를 사용합니다.
    * **구현:** (전체 페이지에 적용)
        * `body` 배경: `#121212`
        * 콘텐츠/테이블 배경: `#1E1E1E`
        * 기본 텍스트 색상: `#E0E0E0`
        * 포인트/활성 메뉴 색상: `#3399FF` (네오스트림 블루 - 가정)
        * 테이블 헤더 텍스트: `#B0B0B0`

3.  **[운영팀] 데이터 밀집형 테이블:**
    * **해석:** 한눈에 많은 정보를 파악할 수 있는 **상태 중심의 테이블**이 필요합니다.
    * **구현:**
        * `full-width` (100%) 테이블을 생성합니다.
        * **테이블 컬럼:** '상태', '썸네일', '영상 제목', '업로더', '요청일시', '액션'
        * **데이터 예시 (3줄):**
            1.  `[Pending]` | (이미지placeholder) | "2025년 4분기 신규 캠페인" | 'PM팀' | '2025-11-13 09:30' | `[승인] [반려]` 버튼
            2.  `[Pending]` | (이미지placeholder) | "신규 입사자 온보딩 영상 (수정)" | 'HR팀' | '2025-11-13 09:15' | `[승인] [반려]` 버튼
            3.  `[Approved]` | (이미지placeholder) | "10월 베스트 콘텐츠 어워드" | '운영팀' | '2025-11-12 16:05' | (버튼 비활성화)
        * **'상태' 컬럼 스타일:**
            * 'Pending' (대기): 노란색 텍스트 (`#FFD700`)
            * 'Approved' (승인): 초록색 텍스트 (`#28A745`)
            * 'Rejected' (반려): 빨간색 텍스트 (`#DC3545`)

---

### [2] 인터랙션 요구사항 (Interactivity)

* **CSS 인터랙션:**
    * 테이블의 각 `row`에 마우스 `hover` 시 `#2A2A2A`로 배경색이 변경되어야 합니다.
    * 사이드바 메뉴 `hover` 시 포인트 색상(` #3399FF`)으로 텍스트 색상이 변경되어야 합니다.
* **JavaScript 인터랙션 (필수):**
    * 테이블의 '액션' 컬럼에 있는 `[승인]` 또는 `[반려]` 버튼을 클릭하면, `alert()` 창으로 **"해당 영상의 제목"**과 **"선택한 액션"**을 표시해야 합니다.
    * (예: "2025년 4분기 신규 캠페인 영상을 '승인' 처리합니다.")
    * `Approved` 상태인 항목의 버튼은 클릭할 수 없도록 `disabled` 처리해야 합니다.

---

### [3] 최종 산출물 (Output)

* **포맷:** **단일 HTML 파일 (`index.html`)**
* **구조:** `<style>` 태그와 `<script>` 태그가 HTML 내부에 모두 포함된 형태여야 합니다. (파일 분리 금지)
* **기타:** 외부 라이브러리(부트스트랩 등)를 사용하지 말고, 순수 HTML/CSS/JS로만 작성하여 속도를 최우선으로 합니다.
